# 5-months-challenge


## Desafio de Desenvolvimento Pessoal e Profissional de 5 Meses para se Tornar Desenvolvedor Pleno Java Backend

![Java Logo](https://upload.wikimedia.org/wikipedia/de/e/e1/Java-Logo.svg)

### Objetivo
Transformar-se em um desenvolvedor pleno de Java Backend em 5 meses, adquirindo tanto habilidades técnicas (hard skills) quanto comportamentais (soft skills) necessárias para o cargo.

### Estrutura Geral
- **Duração Total**: 5 meses
- **Carga Horária Semanal**: 40 horas (6 horas de segunda a sexta, 10 horas aos sábados)
- **Divisão de Tempo**:
  - Hard Skills: 70% (28 horas/semana)
  - Soft Skills: 10% (4 horas/semana)
  - Preparação para Entrevistas: 20% (8 horas/semana)

---

## Programa 1: Hard Skills

### Mês 1: Fundamentos de Java e Programação Orientada a Objetos (POO)
#### Semana 1-2: Fundamentos de Programação e Java
- **Instalação do JDK e Configuração do Ambiente**: Baixar e instalar o JDK. Configurar um IDE como IntelliJ IDEA ou Eclipse.
- **Sintaxe Básica e Estruturas de Controle**: Estudar variáveis, tipos de dados, operadores, loops (for, while) e condições (if, switch).
- **Exercícios Práticos**: Implementar pequenos programas para praticar.
- **Conceitos Básicos de POO**: Estudar classes, objetos, métodos e atributos.
- **Herança, Polimorfismo, Encapsulamento**: Criar hierarquias de classes e usar interfaces.
- **Exercícios Práticos**: Criar um sistema simples (e.g., sistema de gestão de biblioteca).

#### Semana 3-4: Coleções, Generics e Tratamento de Exceções
- **List, Set, Map**: Estudar as interfaces e classes concretas (ArrayList, HashSet, HashMap).
- **Generics**: Compreender e usar tipos genéricos.
- **Exercícios Práticos**: Implementar um programa que use diferentes coleções.
- **Tratamento de Exceções**: Estudar try-catch, throws, exceções personalizadas.
- **Exercícios Práticos**: Manipular exceções em cenários diversos.

### Mês 2: Desenvolvimento Web com Java
#### Semana 5-6: Servlets, JSP e MVC Pattern
- **Ciclo de Vida dos Servlets**: Estudar init(), service(), destroy().
- **JSP e JSTL**: Desenvolver páginas JSP, uso de JSTL.
- **Exercícios Práticos**: Criar uma aplicação web simples.
- **Arquitetura MVC**: Separar responsabilidades em Model, View, Controller.
- **Exercícios Práticos**: Desenvolver uma aplicação MVC básica.

#### Semana 7-8: Spring Framework (Core e MVC)
- **Inversão de Controle (IoC)**: Estudar conceito e uso de @Component, @Autowired.
- **Injeção de Dependência (DI)**: Compreender tipos de injeção (constructor, setter, field).
- **Exercícios Práticos**: Configurar e criar um pequeno projeto Spring.
- **Spring MVC**: Configurar, estudar controllers, views (Thymeleaf).
- **Exercícios Práticos**: Desenvolver uma aplicação web com Spring MVC.

### Mês 3: Bancos de Dados e Persistência
#### Semana 9-10: JDBC e Hibernate
- **Conexão com Banco de Dados**: Configurar e usar o JDBC.
- **Operações CRUD**: Implementar Create, Read, Update, Delete.
- **Exercícios Práticos**: Desenvolver um aplicativo que interaja com um banco de dados.
- **Mapeamento Objeto-Relacional (ORM)**: Estudar conceitos e anotações básicas.
- **Entidades e Relacionamentos**: Mapear entidades e relacionamentos (OneToMany, ManyToOne).
- **Exercícios Práticos**: Criar um sistema utilizando JPA/Hibernate.

#### Semana 11-12: Spring Data JPA e Transações
- **Repositórios**: Usar repositórios Spring Data JPA.
- **Consultas Derivadas e Personalizadas**: Criar queries personalizadas.
- **Exercícios Práticos**: Implementar um sistema CRUD completo com Spring Data JPA.
- **Transações e Gerenciamento de Transações**: Estudar @Transactional, implementar funcionalidades com transações.

### Mês 4: Serviços RESTful e Microsserviços
#### Semana 13-14: RESTful APIs e Documentação
- **Princípios REST**: Estudar HTTP methods, status codes, recursos.
- **Criação de APIs RESTful com Spring Boot**: Configurar e desenvolver APIs.
- **Exercícios Práticos**: Desenvolver uma API RESTful.
- **Documentação com Swagger/OpenAPI**: Configurar o Swagger, documentar automaticamente APIs.

#### Semana 15-16: Microsserviços com Spring Boot e Spring Cloud
- **Arquitetura de Microsserviços**: Estudar conceitos e práticas.
- **Criação de Microsserviços**: Desenvolver microsserviços independentes.
- **Exercícios Práticos**: Implementar um sistema baseado em microsserviços.
- **Spring Cloud**: Configurar, estudar Eureka, Ribbon, Feign, Hystrix.

### Mês 5: Projetos Práticos e Preparação para Entrevistas Técnicas
#### Semana 17-18: Projeto Prático 1
- **Desenvolvimento de uma Aplicação Completa**: Usar Spring Boot, JPA, e Thymeleaf.
- **Teste e Depuração**: Implementar testes unitários com JUnit e Mockito.
- **Exercícios Práticos**: Completar o projeto e realizar testes.

#### Semana 19-20: Projeto Prático 2
- **Desenvolvimento de um Sistema de Gerenciamento de Usuários**: Criar APIs RESTful e documentar.
- **Preparação para Entrevistas Técnicas**: Revisar algoritmos e estruturas de dados, resolver problemas no LeetCode/HackerRank.

## Técnicas de Estudo e Gestão de Tempo

1. **Planejamento Semanal e Diário**
   - **Estabelecer Metas**: Definir metas claras e específicas para cada semana e dia.
   - **Usar Ferramentas de Gestão de Tarefas**: Trello, Asana ou ferramentas similares.

2. **Técnica Pomodoro**
   - **Estudar em Blocos de Tempo**: 25 minutos de foco seguido por 5 minutos de pausa.
   - **Sessões Longas**: Após quatro pomodoros, fazer uma pausa mais longa de 15-30 minutos.

3. **Método GTD (Getting Things Done)**
   - **Coletar e Organizar Tarefas**: Capturar todas as tarefas e ideias, organizar por contexto e prioridade.
   - **Revisão Regular**: Reavaliar tarefas e prioridades semanalmente.

4. **Estudo Ativo**
   - **Prática Regular**: Fazer muitos exercícios práticos.
   - **Aprendizagem Baseada em Projetos**: Aplicar o que aprender em projetos reais.

5. **Grupos de Estudo e Mentoria**
   - **Participar de Grupos de Estudo**: Discutir tópicos com colegas.
   - **Buscar Mentoria**: Encontrar um mentor para orientação.

6. **Revisão Contínua**
   - **Revisão Periódica**: Revisar regularmente o que aprender.
   - **Aplicação Prática**: Testar conhecimentos em novos contextos.
